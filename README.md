# CS 300 Data Structures and Algorithms Portfolio

## Overview
This repository contains selected portfolio artifacts from CS 300 that demonstrate my understanding of data structures, algorithm efficiency, and practical problem solving. The work included here highlights my ability to analyze run-time and memory tradeoffs and implement working code to meet defined requirements.

## Project Summary
The projects in this course focused on managing and organizing course information for a Computer Science program. Project One required analyzing the run-time and memory performance of different data structures. Project Two required implementing a working program that sorts and prints courses in alphanumeric order.

## Problem Solved
The primary problem addressed was determining how to efficiently store, search, and display course data. This required selecting appropriate data structures that balanced performance, memory usage, and maintainability while ensuring the program remained easy to extend.

## Approach and Importance of Data Structures
I approached the problem by evaluating multiple data structures and comparing their strengths and weaknesses based on performance and use case. Understanding data structures is critical because they directly impact how efficiently a program can handle data as it grows. Choosing the right structure allowed the program to perform searches and sorting operations predictably and efficiently.

## Overcoming Challenges
One of the main challenges was translating theoretical concepts such as Big-O notation into practical implementation decisions. I addressed this by breaking the problem into smaller steps, testing different approaches, and validating behavior through incremental development. This process helped ensure correctness while reinforcing core algorithm concepts.

## Impact on Software Design
This project expanded my approach to software design by emphasizing performance considerations early in development. Instead of focusing only on whether a solution works, I now consider how well it works and how it will scale. This mindset leads to more intentional and structured design decisions.

## Writing Maintainable and Adaptable Code
My work in this course improved how I write maintainable and readable code. I focused on clear naming, modular logic, and separation of concerns. Understanding data structures also made it easier to adapt the program to future changes without rewriting core functionality.
